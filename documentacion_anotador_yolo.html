Anotador Manual
de Bounding Boxes

Herramienta de escritorio en Python para dibujar bounding boxes sobre imágenes, asignarles etiquetas y exportar las anotaciones en formato YOLO, con soporte opcional de refinamiento por SAM 3 de Meta.

Python 3.10+ Tkinter · GUI Pillow · Imágenes Ultralytics SAM 3 Exporta YOLO .txt
01

Arquitectura general

El script está organizado en dos capas independientes: una capa funcional con las utilidades de color y el modelo SAM 3, y una capa de interfaz gráfica encapsulada en la clase Annotator.

🎨 Capa de utilidades

Paleta de colores, caché por clase y las dos funciones que integran SAM 3.

🖼 Clase Annotator (GUI)

Hereda de tk.Tk. Gestiona la ventana, el canvas, la lista de boxes y todos los eventos.

📐 Lógica de dibujo

Eventos mousedown/mousemove/mouseup que convierten arrastres en coordenadas reales.

💾 Exportación YOLO

Convierte píxeles a coordenadas normalizadas y genera los tres archivos de salida.

02

Flujo de uso

🚀
1. Arranque Se ejecuta python anotador_yolo.py [imagen]. Si no se pasa imagen, abre el explorador de archivos.
🖼
2. Carga de imagen _load_image() calcula el factor de escala y dibuja la imagen en el canvas con Tkinter.
🖱
3. Dibujo del box El usuario arrastra sobre la imagen. Los eventos de ratón dibujan el rectángulo amarillo temporal.
✍
4. Asignación de etiqueta El usuario escribe el nombre y pulsa Enter o Agregar. _confirm_label() asocia la etiqueta al box.
⚡
5. Refinamiento SAM 3 (opcional) SAM 3 toma el box como prompt y devuelve una máscara de segmentación. El box se ajusta automáticamente al contorno del objeto.
💾
6. Guardado YOLO _save_yolo() normaliza las coordenadas y escribe .txt, classes.txt y .json.

Paleta de colores y caché

Paleta global + función color_for_class 
PALETTE = [
    "#FF4757", "#2ED573", "#1E90FF", "#FF6348", "#ECCC68",
    "#A29BFE", "#00CEC9", "#FD79A8", "#FDCB6E", "#6C5CE7",
    "#FF3F34", "#05C46B", "#575FCF", "#EF5777", "#0be881",
]

_class_color_cache: dict[str, str] = {}

def color_for_class(label: str, all_classes: list[str]) -> str:
    if label not in _class_color_cache:
        idx = all_classes.index(label) if label in all_classes else len(all_classes)
        _class_color_cache[label] = PALETTE[idx % len(PALETTE)]
    return _class_color_cache[label]

PALETTE es una lista de 15 colores hexadecimales de alto contraste, diseñados para ser visibles sobre cualquier tipo de imagen (oscura, clara, saturada).

_class_color_cache es un diccionario global que guarda el color ya calculado para cada etiqueta. Implementa el patrón memoization: la primera vez que se pide el color de "persona" se calcula y se guarda; las siguientes veces se devuelve instantáneamente desde el caché. Esto garantiza consistencia visual: la misma etiqueta siempre tiene el mismo color en toda la sesión.

El operador % módulo en idx % len(PALETTE) hace que si hay más de 15 clases, los colores se reciclen desde el inicio sin provocar un IndexError.

05

Integración con SAM 3

get_sam() — Carga perezosa del modelo

Lazy loading de SAM 3 
_sam_model = None

def get_sam():
    global _sam_model
    if _sam_model is None:
        try:
            from ultralytics import SAM
            print("Cargando SAM 3 … (solo la primera vez)")
            _sam_model = SAM("sam3.pt")
        except Exception as e:
            messagebox.showwarning("SAM 3 no disponible", f"...")
    return _sam_model

Lazy loading significa que SAM 3 (~900 MB) solo se descarga y carga en memoria cuando el usuario hace clic en Refinar SAM3 por primera vez. Si nunca lo usa, la aplicación arranca en menos de 1 segundo.

La palabra clave global es necesaria para modificar la variable declarada fuera de la función. Sin ella, Python crearía una variable local nueva y _sam_model seguiría siendo None en la próxima llamada.

⚠️
El bloque try/except Exception captura cualquier error posible: ultralytics no instalado, sin conexión para descargar los pesos, sin GPU suficiente. En todos los casos la app continúa funcionando para anotación manual.

refine_with_sam() — Refinamiento de máscara

Ajuste del bounding box con máscara SAM 
def refine_with_sam(image_path, x1, y1, x2, y2):
    results = model.predict(
        source=image_path,
        bboxes=[[x1, y1, x2, y2]],  # box manual como prompt
        save=False, verbose=False,
    )
    mask = results[0].masks.data[0].cpu().numpy()
    # Redimensionar máscara al tamaño real de la imagen
    mask_resized = np.array(
        Image.fromarray((mask * 255).astype("uint8"))
            .resize(img.size[::-1])
    )
    ys, xs = np.where(mask_resized > 127)
    return int(xs.min()), int(ys.min()), int(xs.max()), int(ys.max())

SAM 3 devuelve un tensor PyTorch con valores entre 0 y 1 representando la máscara de segmentación. .cpu().numpy() lo convierte a array NumPy (moviendo los datos de GPU a CPU primero si es necesario).

img.size[::-1] invierte (ancho, alto) a (alto, ancho) porque NumPy indexa como filas × columnas mientras Pillow usa ancho × alto.

np.where(mask > 127) devuelve todos los píxeles que pertenecen al objeto. Con xs.min(), ys.min(), xs.max(), ys.max() se calcula el bounding box mínimo que envuelve la máscara completa, resultando en coordenadas más precisas que el box dibujado a mano.

Clase Annotator

Definición de la clase y estado inicial 
class Annotator(tk.Tk):
    MAX_DISPLAY = 900

    def __init__(self, image_path=None):
        super().__init__()

        # Estado de la imagen
        self.image_path = None
        self.orig_img    = None   # Image Pillow original
        self.tk_img      = None   # PhotoImage Tkinter (no GC)
        self.scale       = 1.0   # factor pantalla / real

        # Estado de anotaciones
        self.boxes   = []   # [{label,x1,y1,x2,y2}, ...]
        self.classes = []   # orden de aparición

        # Estado de interacción
        self._drag_start = None
        self._rect_id    = None   # ID del rect. temporal
        self._selected   = None   # índice box seleccionado

Annotator hereda de tk.Tk, que es la ventana raíz de Tkinter. Al heredar, la instancia es la ventana; no hay que crear una ventana por separado.

MAX_DISPLAY = 900 es una constante de clase que limita el tamaño máximo de visualización. Imágenes más grandes se escalan hacia abajo; imágenes pequeñas no se amplían (el tercer argumento 1.0 en min() asegura esto).

self.tk_img debe mantenerse como atributo de instancia. Si fuera una variable local, el recolector de basura de Python la eliminaría y el canvas quedaría completamente en blanco, un error clásico de Tkinter.

self.boxes es la estructura de datos central: una lista de diccionarios donde cada diccionario representa un box con sus coordenadas en píxeles reales (no escaladas), lo que permite guardar correctamente sin importar el zoom de pantalla.

_build_ui() — Construcción de la interfaz

La interfaz se divide en tres zonas construidas con el gestor de layout pack de Tkinter:

🔧
Barra superior (top) Campo de texto, Combobox de autocompletado y botones de acción. Usa side=tk.LEFT para alinear todo horizontalmente.
🖼
Área central (middle) Canvas con scrollbars a la izquierda y Listbox de boxes a la derecha. El Canvas usa fill=tk.BOTH, expand=True para ocupar todo el espacio disponible.
💬
Barra inferior (bot) Mensajes de ayuda contextual que se actualizan dinámicamente con self.hint_var.set().

StringVar y la conexión reactiva

Campo de texto y Combobox compartiendo StringVar 
self.label_var = tk.StringVar()

self.label_entry = tk.Entry(
    top, textvariable=self.label_var, ...
)
self.class_menu = ttk.Combobox(
    top, textvariable=self.label_var, values=[], ...
)

tk.StringVar() es una variable observable de Tkinter. Tanto el Entry (campo de texto) como el Combobox (desplegable) usan la misma label_var, por lo que están sincronizados en tiempo real: si el usuario selecciona una clase del desplegable, aparece en el campo; si escribe en el campo, se actualiza el valor del desplegable. label_var.get() siempre devuelve el valor actual sin importar cuál widget lo modificó.

Canvas con scrollbars

Conexión bidireccional canvas ↔ scrollbars 
self.canvas = tk.Canvas(canvas_frame, cursor="crosshair", ...)
hbar = tk.Scrollbar(canvas_frame, orient=tk.HORIZONTAL,
                    command=self.canvas.xview)
vbar = tk.Scrollbar(canvas_frame, orient=tk.VERTICAL,
                    command=self.canvas.yview)
self.canvas.configure(
    xscrollcommand=hbar.set,
    yscrollcommand=vbar.set
)

La conexión es bidireccional: command=canvas.xview hace que mover el scrollbar desplace el canvas. xscrollcommand=hbar.set hace que desplazar el canvas (con rueda del ratón) actualice la posición del scrollbar. cursor="crosshair" cambia el cursor a una cruz cuando está sobre el canvas, indicando visualmente que se puede dibujar.

_load_image() — Carga y escalado

Cálculo de escala y renderizado en canvas 
def _load_image(self, path):
    self.orig_img = Image.open(path).convert("RGB")
    W, H = self.orig_img.size

    # Escala: reduce si es grande, nunca amplía
    self.scale = min(MAX_DISPLAY / W, MAX_DISPLAY / H, 1.0)

    dw, dh = int(W * self.scale), int(H * self.scale)
    display_img = self.orig_img.resize((dw, dh), Image.LANCZOS)
    self.tk_img = ImageTk.PhotoImage(display_img)

    self.canvas.config(width=dw, height=dh,
                       scrollregion=(0, 0, dw, dh))
    self.canvas.delete("all")
    self.canvas.create_image(0, 0, anchor=tk.NW,
                              image=self.tk_img, tags="bg")

.convert("RGB") normaliza el modo de color: imágenes PNG tienen canal alfa (RGBA), imágenes escaneadas pueden ser escala de grises. Convertir todo a RGB evita errores al mostrar y al procesar con NumPy.

min(MAX/W, MAX/H, 1.0): si la imagen es 2000×1000, el primer valor sería 0.45 y el segundo 0.9; se toma el mínimo (0.45) para que ninguna dimensión supere 900 px. El 1.0 garantiza que imágenes pequeñas no se amplíen.

Image.LANCZOS es el algoritmo de remuestreo de mayor calidad en Pillow. Minimiza el aliasing al reducir resolución, preservando los bordes nítidos necesarios para ver bien los detalles al anotar.

💡
Las coordenadas del box siempre se guardan en píxeles reales (dividiendo por scale al capturar), no en coordenadas de pantalla. Así el guardado YOLO es correcto independientemente del zoom de visualización.

Eventos del ratón — Dibujo del box

Tres eventos encadenados: press → drag → release 
def _on_press(self, event):
    self._drag_start = (event.x, event.y)   # guarda punto inicial
    if self._rect_id:
        self.canvas.delete(self._rect_id)   # limpia rect anterior

def _on_drag(self, event):
    if not self._drag_start: return
    x0, y0 = self._drag_start
    if self._rect_id:
        self.canvas.delete(self._rect_id)
    self._rect_id = self.canvas.create_rectangle(
        x0, y0, event.x, event.y,
        outline="#facc15", width=2, dash=(5, 3),
    )

def _on_release(self, event):
    # Convertir canvas → píxeles reales
    rx0, ry0 = self._canvas_to_real(min(x0,x1), min(y0,y1))
    rx1, ry1 = self._canvas_to_real(max(x0,x1), max(y0,y1))
    self._pending_box = (rx0, ry0, rx1, ry1)

Los tres eventos forman una máquina de estados: mousedown activa el modo de dibujo, mousemove actualiza el rectángulo en tiempo real, mouseup finaliza y convierte las coordenadas.

En _on_drag, el rectángulo anterior se borra y se redibuja en cada evento de movimiento. Esto crea la ilusión de un rectángulo que se estira, aunque técnicamente son cientos de rectángulos dibujados y borrados por segundo.

min/max en _on_release normaliza las coordenadas: el usuario puede arrastrar en cualquier dirección (derecha-abajo, izquierda-arriba, diagonal) y siempre se obtiene un rectángulo con x1 < x2 e y1 < y2.

_canvas_to_real hace simplemente int(coord / self.scale), convirtiendo coordenadas de pantalla a coordenadas reales de la imagen original.

ℹ️
¿Por qué _pending_box? Porque en el momento del mouseup aún no se sabe la etiqueta. El box queda en espera hasta que el usuario la confirma con Enter o el botón Agregar.

Clic derecho — Editar etiqueta

Hit-test para detectar clic sobre un box 
def _on_right_click(self, event):
    rx, ry = self._canvas_to_real(event.x, event.y)
    for i, b in enumerate(self.boxes):
        if b["x1"] <= rx <= b["x2"] and b["y1"] <= ry <= b["y2"]:
            new_lbl = simpledialog.askstring(..., initialvalue=b["label"])
            if new_lbl:
                self.boxes[i]["label"] = new_lbl.strip()
                break

El hit-test (detección de colisión punto-rectángulo) itera sobre todos los boxes y comprueba si el punto del clic está dentro con la condición x1 ≤ rx ≤ x2 AND y1 ≤ ry ≤ y2. Al encontrar el primero que coincide, abre un diálogo de edición y sale del bucle con break.

_confirm_label() y _add_box()

Validación y registro del box 
def _confirm_label(self):
    lbl = self.label_var.get().strip()
    if not lbl:
        messagebox.showwarning(...); return
    if not hasattr(self, "_pending_box") or self._pending_box is None:
        messagebox.showwarning(...); return

    x1, y1, x2, y2 = self._pending_box
    self._pending_box = None   # ← evita confirmar dos veces
    self._add_box(lbl, x1, y1, x2, y2)

def _add_box(self, label, x1, y1, x2, y2):
    if label not in self.classes:
        self.classes.append(label)
        self.class_menu["values"] = self.classes
    self.boxes.append({"label": label,
                       "x1": x1, "y1": y1,
                       "x2": x2, "y2": y2})
    self._redraw_boxes()
    self._refresh_list()

hasattr(self, "_pending_box") verifica que el atributo exista, porque la primera vez que el usuario pulsa Enter sin haber dibujado nada, _pending_box aún no se ha creado como atributo.

Poner _pending_box = None inmediatamente después de leerlo es un patrón de seguridad: si el usuario pulsa Enter dos veces rápido, la segunda llamada encontrará None y mostrará el aviso en lugar de agregar el mismo box dos veces.

En _add_box, la clase solo se agrega a self.classes si es nueva (con not in), preservando el orden de aparición que YOLO necesita para mantener consistentes los índices.

_redraw_boxes() — Renderizado

Borrar y redibujar todos los boxes desde cero 
def _redraw_boxes(self):
    self.canvas.delete("box")   # solo borra boxes, no la imagen
    for i, b in enumerate(self.boxes):
        col = color_for_class(b["label"], self.classes)
        selected = (i == self._selected)

        x1 = b["x1"] * self.scale   # real → pantalla
        y1 = b["y1"] * self.scale

        self.canvas.create_rectangle(
            x1, y1, x2, y2,
            outline=col,
            width=3 if selected else 2,
            dash=() if not selected else (8, 4),
            tags="box",
        )

canvas.delete("box") borra solo los elementos etiquetados con "box", sin tocar la imagen de fondo etiquetada como "bg". Los tags de Tkinter permiten gestionar grupos de elementos del canvas de forma selectiva.

Las coordenadas se multiplican por self.scale para pasar de píxeles reales a coordenadas de pantalla. Esto es la operación inversa a _canvas_to_real.

El box seleccionado tiene borde más grueso (width=3) y línea punteada (dash=(8,4)): 8 px pintados, 4 px vacíos. El box normal tiene dash=(), que significa línea continua.

_save_yolo() — Exportación

Conversión de píxeles a formato YOLO normalizado 
def _save_yolo(self):
    W, H = self.orig_img.size
    lines = []
    for b in self.boxes:
        cid = self.classes.index(b["label"])

        cx = round((b["x1"] + b["x2"]) / 2 / W, 6)  # centro X norm.
        cy = round((b["y1"] + b["y2"]) / 2 / H, 6)  # centro Y norm.
        bw = round((b["x2"] - b["x1"]) / W, 6)       # ancho norm.
        bh = round((b["y2"] - b["y1"]) / H, 6)       # alto norm.

        lines.append(f"{cid} {cx} {cy} {bw} {bh}")

YOLO requiere que todas las coordenadas estén normalizadas entre 0 y 1 y usa el centro del objeto en lugar de las esquinas. Esta tabla resume la conversión:

Campo YOLOFórmulaDescripción
clase_idclasses.index(label)Índice entero de la clase en classes.txt.
cx(x1+x2)/2 / WCentro X normalizado. Rango [0,1].
cy(y1+y2)/2 / HCentro Y normalizado. Rango [0,1].
bw(x2-x1) / WAncho del box normalizado.
bh(y2-y1) / HAlto del box normalizado.

Tres archivos de salida

📄
imagen.txt Una línea por box: clase_id cx cy bw bh. Es el archivo de anotaciones que YOLO lee durante el entrenamiento.
📋
classes.txt Una clase por línea, en el mismo orden que los índices. YOLO lo usa para saber qué nombre corresponde a cada número.
🗂
imagen_annotations.json Respaldo completo con coordenadas en píxeles reales, tamaño de imagen y lista de clases. Útil para reimportar o convertir a otros formatos.

Entry point — Arranque del programa

Bloque principal 
if __name__ == "__main__":
    img_path = sys.argv[1] if len(sys.argv) > 1 else None
    app = Annotator(image_path=img_path)
    app.mainloop()

if __name__ == "__main__" es el guardián de ejecución de Python. Cuando el archivo se ejecuta directamente (python anotador_yolo.py), la variable __name__ vale "__main__" y el bloque se ejecuta. Si otro script importara este archivo como módulo (import anotador_yolo), __name__ valdría "anotador_yolo" y el bloque no se ejecutaría, evitando que la ventana se abra al importar.

sys.argv es la lista de argumentos de línea de comandos. sys.argv[0] es siempre el nombre del script. sys.argv[1] sería la ruta de imagen si se ejecutó como python anotador_yolo.py foto.jpg. len(sys.argv) > 1 comprueba si se pasó ese argumento antes de leerlo.

app.mainloop() inicia el bucle de eventos de Tkinter, que mantiene la ventana activa y procesa todos los eventos (clics, teclas, redraws) hasta que el usuario cierra la ventana o ejecuta self.destroy().
